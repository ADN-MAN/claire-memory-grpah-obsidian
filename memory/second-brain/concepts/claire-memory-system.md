# Claire Memory System

Tags: #claire #memory #system

Claire uses memory in layered form.

## Layers

### 1. Doctrine
Stable identity and operating logic live in:
- [[../../Claire Genesis]]
- [[../../identity/SOUL]]
- [[../../identity/AGENTS]]
- [[../../MEMORY]]

### 2. Structured knowledge
Reusable, linked knowledge lives in this second brain.
Use it for:
- concepts
- entities
- maps
- structured summaries
- operating playbooks

### 3. Daily operational memory
Chronological notes and raw continuity live in:
- `../../daily/`

## Promotion flow
Typical movement should be:

Daily memory -> second brain -> doctrine memory

That means:
- capture first
- structure second
- distill last

## Rule of quality
A good memory system reduces repeated explanation, supports continuity, and preserves signal.
A bad memory system becomes a dump that weakens retrieval.
